# 局限性声明（Limitations）

本评测结论受以下边界约束，读者应据此判断外推有效性。

## 1. 样本局限

- 单样本（`tinyexpr`）不足以代表 C 生态全貌。
- 样本规模小、依赖少、代码风格整洁，难度低于典型生产代码。
- 不覆盖重宏、并发、I/O、平台分支密集等高复杂度场景。

## 2. 实验者局限

- 单实验者执行，评分存在主观偏差。
- 实验者为 C/Rust 学习者，可能高估模型解释质量或低估惯用法偏差。
- 人机交互风格对结果有显著影响，跨实验者可复现性有限。

## 3. 工具与环境局限

- 结论绑定于实验期 Claude Code/模型版本。
- 环境为 Windows + MinGW + GNU Rust toolchain；跨平台结果可能不同。

## 4. 方法学局限

- 成功标准以“编译通过 + 对照通过”为主，未系统评估性能与长期可维护性。
- 对照测试集由人工设计，仍可能遗漏对抗性输入和稀有边界。
- 定性指标（如“惯用性”）无法完全消除评判者主观性。

## 5. 适用范围

较适用：中小规模、算法型、依赖简单、可构建对照测试的 C 项目。  
不宜外推：10K+ LOC、大量宏、系统编程、强性能约束或高并发项目。

## 6. 建议的后续验证

- 多样本、多实验者重复实验。
- 横向比较不同模型与工具链。
- 引入性能基准、fuzz 与盲测评审。
