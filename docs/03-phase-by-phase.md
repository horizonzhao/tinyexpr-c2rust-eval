# 03. 五阶段执行记录（Phase by Phase）

本文按固定阶段记录实验进展与关键决策，不展开技术细节（细节见 `04-deep-dive/`）。

## 阶段 1：理解 C 实现

- 目标：建立模块与调用关系认知。
- 完成标准：可独立解释词法、语法、求值主流程。
- 记录要点：理解障碍、关键转折、模型解释有效性。

## 阶段 2：搭建 Rust 骨架

- 目标：完成类型定义与函数签名，保证 `cargo build` 通过。
- 完成标准：主体接口齐备，函数体可暂用占位实现。
- 记录要点：`te_expr` 映射方案、生命周期初步设计。

## 阶段 3：逐函数翻译

- 目标：替换全部占位实现并补齐单元测试。
- 完成标准：核心函数均有对应测试且通过。
- 记录要点：每个函数的修正轮次、人工介入类型、失败模式。

## 阶段 4：C/Rust 对照验证

- 目标：对照测试收敛到可接受差异阈值。
- 完成标准：`scripts/compare.ps1` 全量通过或差异可解释。
- 记录要点：浮点策略、边界输入、语义偏移来源。

## 阶段 5：报告整理

- 目标：将过程记录整理为结构化结论。
- 完成标准：`FINDINGS.md`、`CONCLUSIONS.md`、`LIMITATIONS.md` 完整一致。
- 记录要点：证据链闭环、可复现性说明、未解问题。
