# 01. 实验背景（Background）

本项目关注一个实践问题：LLM 驱动的编程代理能否在可控人工监督下完成 C→Rust 迁移，并保持语义一致。

## 1. 研究动机

- C 代码迁移到 Rust 的主要障碍是重写成本与语义风险。
- 通用代码助手在“可解释翻译”场景中的能力仍缺少系统性证据。
- 公开评测常偏向函数级生成，缺少真实项目翻译的过程数据。

## 2. 研究定位

本实验采用单项目深描：选择 `tinyexpr`，记录完整交互、测试与修正过程，重点分析：

- 可自动化的翻译部分；
- 必须人工决策的语义重构部分；
- 失败模式及其可复用规避策略。

## 3. 与既有工作的关系

- 相比规则转译器评测，本实验强调“人机协作 + 惯用 Rust 产出”。
- 相比通用 benchmark，本实验强调“真实仓库过程可观察性”。
