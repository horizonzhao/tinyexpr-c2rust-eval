# 深度文章：浮点语义差异（Float Semantics）

本章讨论 C 与 Rust 在浮点计算上的一致性风险及其验证策略。

## 1. 风险来源

- 标准库实现差异可能导致末位偏移。
- 编译优化路径差异可能改变中间舍入。
- NaN/Inf 表示与比较规则在边界场景下不完全等价。

## 2. 验证策略

| 策略 | 适用场景 | 备注 |
|---|---|---|
| 位级相等 | 纯算术与可控路径 | 最严格 |
| 相对误差阈值 | libm 函数输出 | 需处理接近 0 的分母 |
| ULP 阈值 | 高精度一致性分析 | 解释性更强 |

## 3. 报告要求

所有不一致用例需记录：

- 输入表达式
- C 与 Rust 输出
- 使用的容差判据
- 是否可归因于实现差异而非翻译错误

## 4. 工程建议

- 在对照脚本中显式声明容差策略，避免隐式判断。
- 对数学库调用建立专项回归用例。
- 将“可解释偏差”与“语义错误”分开统计。
