# 深度文章：联合体到枚举映射（Union → Enum）

本章聚焦 `te_expr` 的数据建模迁移。

## 1. 问题定义

C 端采用 `type + union + 弹性数组`，Rust 端需在三者间重新平衡：

- 类型安全
- 内存布局
- 实现复杂度

## 2. 方案比较

| 方案 | 优势 | 风险 |
|---|---|---|
| `enum` 建模 | 类型安全高，匹配清晰 | 可能增大结构体体积 |
| 保留 `union` | 接近 C 布局 | `unsafe` 成本高 |
| 扁平字段 | 实现快 | 语义约束弱，易失配 |

## 3. 选择依据

优先保证语义可验证与维护性；仅在有实测性能瓶颈时考虑低层布局优化。

## 4. 关联议题

- `parameters[1]` 映射到 `Vec` 或切片容器策略
- 变量绑定与函数指针的类型表达
- `unsafe` 使用边界与审计记录
