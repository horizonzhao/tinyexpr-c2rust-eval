# 深度文章：解析器翻译（Parser Translation）

本章分析递归下降解析器在 C→Rust 迁移中的关键决策。

## 1. 核心难点

- C 端游标推进依赖指针语义，Rust 需显式状态建模。
- C 端错误传播多为哨兵值，Rust 更适合 `Result`。
- 递归调用与可变借用组合容易触发借用约束冲突。

## 2. 决策框架

| 议题 | 候选方案 | 评估标准 |
|---|---|---|
| 游标表示 | `&mut &str` / 索引 / `Lexer` | 可读性、回溯成本、借用复杂度 |
| 错误传播 | 哨兵值 / `Option` / `Result` | 可诊断性、接口一致性 |
| 语法层实现 | 直译层级函数 / Pratt | 语义可核验性、改造成本 |

## 3. 记录要求

每次解析器改动需记录：

- 变更前后行为差异；
- 相关测试用例与结果；
- 是否触发新的所有权或生命周期成本。

## 4. 输出目标

本章最终应回答：在保证语义一致前提下，何种解析器设计在 Rust 中具有最低维护成本。
