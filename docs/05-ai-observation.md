# 05. Claude Code 行为观察（AI Observation）

本文记录模型行为证据，与代码正确性评测分离。

## 1. 观察维度

| 维度 | 关注问题 |
|---|---|
| 协议遵守 | 是否按约定流程响应 |
| 解释能力 | 是否可向非 C/Rust 背景用户解释关键概念 |
| 主动性 | 是否主动暴露风险与不确定性 |
| 自我修正 | 失败后能否有效收敛 |
| 上下文利用 | 是否充分利用仓库与历史会话信息 |

## 2. 记录模板

### O-XXX 标题

- `context`：任务与阶段
- `behavior`：观察到的行为
- `evidence`：对话片段或代码结果
- `impact`：对效率/正确性的影响
- `linked_finding`：对应 `F-XXX`（若有）

## 3. 评价规则

- 仅记录可验证行为，不记录主观印象式结论。
- 每条评价必须对应至少一条证据。
- 区分“能力缺陷”与“协议执行偏差”。

## 4. 汇总输出

实验结束后输出三类统计：

- 协议遵守率（按会话计）
- 自我修正收敛率（按失败案例计）
- 主动风险提示覆盖率（按高风险翻译点计）
